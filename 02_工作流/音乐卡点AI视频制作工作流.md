# 音乐卡点 AI 视频制作工作流

> [!important] 核心原则
> 卡点视频先锁音乐事件与输出帧，再制作画面。自动节拍检测只能提供候选点，不能代替人耳确认歌词、强音、鼓点和情绪落点。人物锚点、单镜关键帧、转场关键帧与最终同框图全部审核并下载落盘前，不提交视频生成任务。

## 适用范围

适用于需要人物连续性、动作转场、歌词强音揭示、多人同框、推拉运镜或抖音式音乐卡点的 AI 视频。通用要求同时遵循 [[AI视频制作工作流]]。

## 一、开工输入

开始前确认并留档：

- 音乐或参考视频的本地文件、使用权限与目标片段。
- 目标时长、画幅、分辨率、输出帧率和发布平台。
- 人物锚点组、本项目选定锚点及其版本。
- 角色数量、服装、道具、场景、最终同框关系。
- 需要卡住的歌词、拟声词、鼓点、重拍或情绪爆点。
- 生成入口、实时模型、预算与用户审批。

> [!warning] 不允许的起步方式
> 未确认音乐中的关键歌词或强音，只看 BPM 或波形就开始生成；未拿齐关键帧就批量生成视频；把模型生成的临时链接当成已经拥有的正式资产。

## 二、卡点位置先行

### 2.1 建立候选卡点

1. 从参考视频提取一条连续原始音轨，不拼接模型自带音乐。
2. 使用波形、频谱、onset/beat detection、BPM 和场景参考生成候选时间。
3. 单独循环播放候选点前后约 0.5～1 秒，人工确认：
   - 歌词或拟声词真正开始的瞬间；
   - 重鼓、强拍或音色爆发的瞬间；
   - 视觉事件应该发生在起音、峰值还是尾音；
   - 音乐最强点是否与自动检测结果一致。
4. 若用户指定“某一句”“嘿耶”“最强音”等语义点，以人耳和用户确认结果为准，不能自行替换成附近更强的鼓点。

### 2.2 建立卡点表

每个关键点必须记录：

| 字段 | 说明 |
|---|---|
| `cue_name` | 歌词、拟声词、鼓点或事件名称 |
| `candidate_time` | 自动分析候选时间 |
| `confirmed_time` | 人耳确认的源音频时间 |
| `source` | 人耳、歌词、波形、onset、参考视频剪辑点等证据 |
| `target_fps` | 输出帧率 |
| `target_frame` | 量化后的输出帧 |
| `render_time` | 该帧在输出中的时间 |
| `error_ms` | 与确认时间的误差 |
| `visual_event` | 该帧必须出现的画面事件 |
| `transition_rule` | 转场必须在该帧前结束还是从该帧开始 |

### 2.3 时间量化为帧

使用固定公式：

```text
target_frame = round(confirmed_time × fps)
render_time = target_frame ÷ fps
error_ms = (render_time - confirmed_time) × 1000
```

- 30 fps 的正常量化误差上限为半帧，即约 16.7 ms。
- 所有剪辑计划以帧号为主，秒数只用于说明。
- 强音揭示规则：若要求“强音一响就同框”，则同框的第一帧必须等于 `target_frame`；白闪、挥袖、遮挡等过渡必须在前一帧结束，不得覆盖或推迟同框第一帧。

### 2.4 卡点关卡

在图片或视频生成前，用户至少确认：

- [ ] 关键歌词/强音名称正确。
- [ ] 人耳确认时间正确。
- [ ] 每个强音对应的视觉事件正确。
- [ ] 最终同框、变装或高潮揭示的目标帧正确。
- [ ] 不需要的快速蒙太奇已经删除。

## 三、图片资产先行

### 3.1 图片路由

高保真人物、多参考图融合、服装/场景统一和多人同框图可优先使用 **IMA Studio 的 Nano Banana Pro**。正式提交前必须查询 IMA 当前产品列表，确认：

- 当前准确 `model_id`，不得仅凭历史记录推断；
- 文生图或图生图模式；
- 参考图数量、画幅、分辨率与输出格式；
- 单任务积分、任务数、总积分与账户余额；
- 用户已确认本次积分消耗。

历史上曾使用 `gemini-3-pro-image`，但模型 ID、积分和能力属于动态信息，不写成永久固定值。

### 3.2 参考图职责表

生成前先写清每张参考图只负责什么：

| 优先级 | 资产 | 允许控制 | 禁止继承 |
|---:|---|---|---|
| 1 | 唯一身份主锚点 | 脸型、五官比例、人物辨识度 | 错误年龄、疲惫皮肤、旧服装或旧背景 |
| 2 | 状态参考 | 年龄、皮肤、表情强度、健康状态 | 覆盖或平均主锚点身份 |
| 3 | 服装/道具参考 | 款式、颜色、材质、佩戴关系 | 参考图人物的脸 |
| 4 | 场景/构图参考 | 空间、机位、光线、色彩、人物位置 | 改变身份与服装 |

提示词必须明确“主锚点是唯一最高优先级身份来源，辅助图的人脸无效，不得混脸”。

### 3.3 必须拿齐的图片

视频阶段开始前至少具备：

- 唯一人物主锚点。
- 每个角色/造型的批准关键帧。
- 需要动作擦镜时的方向与手别设计图或说明。
- 最终同框图，提前解决人数、遮挡、站位和服装串色。
- 必要的首帧/尾帧或转场锚点。

### 3.4 图片审核与落盘

每张图片都必须：

1. 下载到项目目录，不能只保存临时 URL。
2. 校验文件签名、格式、宽高、画幅和可读取性。
3. 与主锚点对照身份、年龄、皮肤、左右眼、手部和体型。
4. 检查服装、道具、光线、白平衡、空间和物理常识。
5. 制作总览联系表和相邻镜头对照图。
6. 标记“通过 / 需修 / 废弃”；只有“通过”图可进入视频生成。
7. 保存模型、提示词、参考图顺序与职责、任务 ID、积分和本地路径。

> [!warning] 图片关卡
> 主锚点、全部单镜关键帧和最终同框图未全部落盘并通过审核时，禁止为了赶进度先提交视频。

## 四、视频生成

### 4.1 路由与付费关卡

- 单张批准首帧生成自然动作、人物/场景一致性：优先评估 HappyHorse 1.1。
- 首尾帧精确控制、视频续写、自定义音频或运镜复刻：优先评估 Wan 2.7。
- 每次提交前查询当前模型、时长、分辨率、参考输入、价格和余额，并取得用户确认。
- 每个镜头只提交一次，立即保存任务 ID；等待或网络超时只轮询原任务，不重复提交。

### 4.2 单镜头设计

每个镜头只安排一个主要动作，明确：

- 起势、执行、落点、余韵和转场遮挡发生的时间。
- 相机方向、速度、焦段、景别和最终构图。
- 擦镜物体、屏幕方向、人物实际手别与另一只手的位置。
- 与前后镜头是否交替方向，避免连续重复同一只手、同一袖子方向或同一构图。
- 身份、服装、道具、光线、表情和禁止项。

手别是高风险指令。提示词写“人物左手/右手”之外，还要同时写“画面左侧/右侧、运动起点、运动终点、另一只手必须留在哪里”。

### 4.3 结果下载与单镜验收

下载后为每个镜头制作首帧、中帧、尾帧和逐帧联系表，检查：

- 人物身份、年龄、皮肤和表情是否稳定。
- 实际使用的手、挥袖方向与相机运动是否符合设计。
- 另一只手、道具和衣料是否变形、复制或穿模。
- 擦镜是否真的覆盖镜头，覆盖发生在何时。
- 结尾是否可作为下一镜头的剪辑依据。

模型没有遵循手别时不得口头宣称通过。可先评估完整镜头统一镜像是否能修正屏幕侧与方向；若镜像会破坏文字、道具、脸部非对称或空间逻辑，则只补做该问题镜头，并重新走付费确认。

## 五、按帧剪辑

### 5.1 技术母版

先统一输出：

- 固定分辨率、帧率、像素格式和编码。
- 所有片段先缩放/裁切到同一画布。
- 片段变速后按目标帧数再次 `trim`，不足时只允许极短自然补帧或尾帧保持。
- 音乐使用一条连续整轨，不因换镜重启、断拍或换响度。

### 5.2 剪辑原则

- 主剪辑点直接使用卡点表的 `target_frame`。
- 转场动作可以提前起势，但必须在目标帧之前完成。
- 强音硬切可以成立，但若动作已停止、景别跳变过大，则在强音前加入有动机的挥袖、遮挡、白闪或构图匹配；不能把同框推迟到强音之后。
- 快速蒙太奇只有在用户明确需要且每个短镜都服务音乐时使用；不得为了显得“卡点多”擅自插入。
- 最终推拉运镜方向在生成前写入分镜；现有镜头反向播放可把拉远改为拉近，但必须审核人物呼吸、眨眼、衣料和道具的逆向感。

## 六、成片卡点反查

成片导出后必须反向证明“卡对了”：

1. 使用媒体探测工具核验时长、宽高、帧率、总帧数、视频编码、音频采样率和声道。
2. 用场景变化检测或逐帧检查得到成片的实际切点。
3. 将实际切点与卡点表逐项比较，记录误差；不能只看剪辑脚本声称的时间。
4. 为每个关键点制作“目标帧前 5～10 帧 + 目标帧后 5～10 帧”的逐帧图。
5. 人耳完整播放一次，再循环播放每个歌词/强音前后约 1 秒。
6. 检查黑帧、闪帧、重复帧、卡顿、音乐断点和突然变响。
7. 用户指定的强音画面若仍晚一帧或早一拍，直接判为“需修”，不得用“附近也是强拍”解释。

### 允许误差

- 视觉硬切：优先落在最近输出帧，误差不超过半帧。
- 有意设计的预闪/遮挡：只能发生在目标帧之前；目标帧必须已经是承诺的主体画面。
- 如果平台重编码改变帧率，先导出平台测试版并再次反查。

## 八、失败教训

- 自动检测到“最强鼓点”不代表用户说的歌词强音；语义点必须人耳确认。
- 不应先生成视频、后补最终同框图；多人同框问题应在图片阶段解决。
- 用户要求“强音直接同框”时，不在强音前后擅加快速动作蒙太奇。
- 白衣或其他镜头动作已经结束时直接跳大全景会生硬；让有动机的袖子/遮挡在强音前完成，再于强音第一帧揭示。
- 水平镜像主要改变屏幕方向，不能替代手别审核；必须看逐帧图确认哪只手实际发起。
- 提示词即使明确左右手，模型仍可能不服从；审核结果比提示词文本更可信。
- 任务轮询中断不代表生成失败；保存任务 ID，只恢复轮询，禁止重复提交。

## 九、交付清单

- [ ] 音乐文件与使用说明。
- [ ] 已确认卡点表，含歌词/强音、帧号和误差。
- [ ] 主锚点、所有关键帧和最终同框图均已下载并通过审核。
- [ ] IMA 图片任务的模型、积分、任务 ID、提示词和本地路径已记录。
- [ ] 视频任务的模型、费用、任务 ID、状态和本地路径已记录。
- [ ] 每个镜头的首中尾帧、手别、方向和转场已审核。
- [ ] 实际场景切点已与卡点表反查。
- [ ] 关键强音逐帧图证明目标帧已经出现承诺画面。
- [ ] 成片规格、黑帧、音轨连续性和正常速度播放通过。
- [ ] 只交付结论为“通过”的版本。
