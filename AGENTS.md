# Zcode 跨项目记忆指令

## 任务开始

1. 每次新任务先读取 `/Users/wangzirui/Zcode工作流汇总/00_HOME/Zcode启动必读.md`。
2. 再读取 `/Users/wangzirui/Zcode工作流汇总/01_长期记忆/用户长期偏好.md`。
3. 根据当前任务主题，只读取相关的项目、工作流、决策和资产笔记；不每次扫描整个仓库。

## 任务结束

1. 只把用户已确认、能跨项目复用的信息写入长期记忆。
2. 临时状态、未确认推断和一次性指令只写项目记录或每日复盘。
3. 不写入 API Key、密码、会话令牌或不必要的敏感信息。
4. 冲突信息以用户最新明确确认为准，旧版移入归档。

## 记忆库写入后强制核验

每次新增或修改 `/Users/wangzirui/Zcode工作流汇总` 中的内容后，必须立即核验：

1. 文件归类与目录正确。
2. 正式笔记的 YAML frontmatter、标题、日期和分层标签正确。
3. 新标签已写入 `00_HOME/标签索引.md`，无重复近义标签。
4. 新的核心内容已写入 `00_HOME/内容索引.md` 或对应分类索引。
5. Obsidian 双链无无意义断链，Markdown 和 JSON 可正常解析。
6. 新旧记忆无未处理冲突，旧版必要时已归档。
7. 内容中无 API Key、密码、会话令牌或不必要的敏感信息。
8. 交付时回报标签、索引和断链核验结果。

任何一项未通过时，不得宣称记忆库更新已完成。

可通过 `python3 /Users/wangzirui/Zcode工作流汇总/99_System/validate_vault.py` 执行自动核验，输出 `PASS` 才算通过。
