# DEC-2026-07-17：建立 Zcode 跨项目永久记忆库

## 背景

Zcode 需要在不同项目和对话之间保留已确认的长期规划、用户偏好、工作流、项目状态和关键决策。用户已有 Codex 仓库作为参考，要求 Zcode 建立独立仓库，借鉴结构与规则，但与 Codex 隔离。

## 决策

- 使用 Obsidian 仓库 `/Users/wangzirui/Zcode工作流汇总` 作为跨项目记忆库。
- Zcode 每次任务先读取精简启动页，再按需读取相关笔记。
- 不在记忆库中保存密钥、密码或会话令牌。
- 每次新增或修改记忆库后，必须核验归类、属性与标签、标签索引、内容索引、双链、格式、冲突与敏感信息；核验不通过不得宣称完成。
- 通过 `99_System/validate_vault.py` 执行自动核验，输出 `PASS` 才算通过。
- 全局指令写入 `~/.zcode/AGENTS.md`（用户级默认指令，对每个工作区自动生效），指向本仓库的启动必读与长期偏好，实现"每次启动自动读"。
- 丁美霞数字人资产、状态参考体系与 OpenRouter 视频工作流从 Codex 仓库复制并改造（改 Codex→Zcode、路径与双链适配），作为 Zcode 处理同类视频任务的起点。

## 理由

- Obsidian Markdown 便于用户直接阅读、修改和溯源。
- 分层读取可以避免每次扫描全库的速度和上下文浪费。
- 决策、工作流、项目和每日记录分开，可降低过期信息污染长期记忆的风险。
- 与 Codex 隔离可避免混用项目事实；丁美霞资产作为已确认的跨项目复用素材，经用户确认后分别写入两个仓库。

## 影响

- 长期记忆只存稳定事实，短期过程存入每日复盘或项目页。
- 任何与新确认信息冲突的旧记忆应归档，不并行生效。
- 每日复盘不建立系统级自动任务（无 launchd / cron），采用"从 Codex 抄一份并改造自验"的方式，在次日首次任务开始时补做。
