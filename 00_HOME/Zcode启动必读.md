# Zcode 启动必读

## 每次任务的读取顺序

1. **先检查是否需要补复盘**（不可跳过）：运行 `date +%Y-%m-%d` 取今天日期，列出 `05_每日复盘/` 下所有 `YYYY-MM-DD.md`，找最大日期。
   - 最大日期 < 今天且 ≥ 今天-7：按 [[../05_每日复盘/README|复盘说明]] 的"生成方式"从 Codex 抄录并改造前一日复盘，跑 `validate_vault.py` 通过后再继续本任务。
   - 最大日期 = 今天：已复盘，跳过。
   - 今天-7 内无任何复盘文件（新库或长期未用）：不强制补，继续任务。
   - 需补的复盘超过 3 天：不逐日补，只在当日复盘里说明"X 日至 Y 日未复盘"。
2. 先读本页。
3. 读取 [[../01_长期记忆/用户长期偏好]]。
4. 根据任务选择一个相关工作流或项目页，不全库扫描。
5. 任务结束后，只把已确认、长期有用的信息写回记忆库。

## 默认行为

- 使用中文沟通，先说结果，表达清楚直接。
- 能安全继续时主动执行；涉及付费、外部发送、上传或高影响操作时先确认权限与成本。
- 不把短期指令、临时状态或未经确认的推断写成长期偏好。
- 不记录 API Key、密码、身份证号、会话令牌等敏感信息。
- 冲突信息以用户最新明确确认为准；旧信息移入归档，不直接删除历史。

## 记忆写入标准

只有满足下列任一条才写入长期记忆：

- 用户明确说"记住"、"以后都"或"作为最终版"。
- 同一偏好在不同任务中多次出现。
- 该信息会直接影响后续项目的正确性、成本或风险。

## 任务结束写回

- 新长期偏好 → [[../01_长期记忆/用户长期偏好]]
- 新环境信息 → [[../01_长期记忆/工具与环境注意事项]]
- 可复用方法 → `02_工作流`
- 关键决策 → `03_决策记录`
- 项目状态 → `04_项目`
- 当日摘要 → `05_每日复盘`

## 记忆库每次写入后强制核验

每次新增或修改记忆库内容后，必须立即完成以下核验；任何一项未通过时，不得宣称任务已完成：

1. **归类核验**：文件放在正确目录，长期记忆、项目过程和当日复盘不混用。
2. **属性与标签核验**：正式笔记具有合法 YAML frontmatter、`title`、日期和合适的分层标签。
3. **标签索引核验**：新标签已登记到 [[标签索引]]，不创建含义重复的近义标签。
4. **内容索引核验**：新增的核心笔记、工作流、决策和项目已加入 [[内容索引]] 或对应分类索引。
5. **链接核验**：检查新增或受影响的 Obsidian 双链，不得留下无意义的断链。
6. **格式核验**：Markdown 可正常渲染；Obsidian JSON 配置能被正常解析。
7. **冲突与时效核验**：新记忆与已有记忆冲突时，以用户最新明确确认为准，旧版移入归档。
8. **安全核验**：不得出现 API Key、密码、会话令牌或不必要的个人敏感信息。
9. **结果回报**：交付时说明新增/修改的文件、标签情况、索引情况和断链检查结果。
