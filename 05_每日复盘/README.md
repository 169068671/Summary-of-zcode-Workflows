# 每日复盘

## 生成方式

不建立系统级自动任务（无 launchd / cron）。采用"从 Codex 抄一份"的方式：

1. 读取 `/Users/wangzirui/Codex工作流汇总/05_每日复盘/YYYY-MM-DD.md`。
2. 复制到本目录同名文件 `YYYY-MM-DD.md`。
3. 把正文中的 "Codex" 替换为 "Zcode"，去掉 Codex 专属的项目事实（如 Codex 建库记录），保留通用的完成项、失败经验和长期记忆候选。
4. 如果 Zcode 当日有 Codex 未覆盖的独立任务，手动补充到对应章节。
5. 如果 Codex 当日无复盘文件，从 [[../06_模板/每日复盘模板]] 新建，标注"当日无 Codex 复盘可抄"。

## 执行时机

在次日首次任务开始时补做前一日复盘。

## 处理规则

1. 列出当日完成的任务、交付文件和未完成事项。
2. 记录失败原因、修复方法、任务 ID 和实际成本，但不保存密钥。
3. 区分"当日过程"与"长期记忆候选"。
4. 只把已确认的稳定信息升级到 `01_长期记忆`、`02_工作流`、`03_决策记录` 或 `04_项目`。
5. 没有新项目时也生成简短记录，标注"当日无新增项目记录"。
6. 写入完成后执行 [[../00_HOME/Zcode启动必读#记忆库每次写入后强制核验|记忆库强制核验]]，检查标签、索引、链接、格式、冲突与敏感信息。

## 抄录后强制自验

抄写并改造完复盘后，必须运行：

```bash
python3 /Users/wangzirui/Zcode工作流汇总/99_System/validate_vault.py
```

输出 `PASS` 才算完成。常见需修复项：

- 正文残留 "Codex" 未替换 → 全文搜索替换。
- 残留 Codex 专属路径（如 `/Users/wangzirui/Codex工作流汇总/...`）→ 删除或改为 Zcode 对应路径。
- 新增标签未登记 → 补到 [[../00_HOME/标签索引]]。
- 双链指向 Codex 仓库的笔记 → 改为 Zcode 对应笔记或删除。

## 文件命名

`YYYY-MM-DD.md`
